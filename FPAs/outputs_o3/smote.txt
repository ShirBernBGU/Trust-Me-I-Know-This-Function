import random
import math

def smote(minority_samples, N=100, k=5):
    """
    Synthetic Minority Over-sampling Technique (SMOTE).

    Parameters:
        minority_samples (list of list/tuple): Existing minority class samples.
        N (int): Percentage of over-sampling (must be a multiple of 100).
                 For example, N=200 will generate 2 new samples for each original one.
        k (int): Number of nearest neighbors to consider.

    Returns:
        list of list: Newly generated synthetic samples.
    """
    if not minority_samples:
        return []
    if N % 100 != 0 or N <= 0:
        raise ValueError("N must be a positive multiple of 100.")

    def euclidean(a, b):
        return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))

    n_new_samples_per_point = N // 100
    synthetic = []
    for i, sample in enumerate(minority_samples):
        # Compute distances to all other samples
        distances = sorted(
            (euclidean(sample, other), other)
            for j, other in enumerate(minority_samples) if j != i
        )
        # Select k nearest neighbors
        neighbors = [neighbor for _, neighbor in distances[:k]]

        # Generate synthetic samples
        for _ in range(n_new_samples_per_point):
            neighbor = random.choice(neighbors)
            gap = random.random()
            synthetic.append([
                sample[attr] + gap * (neighbor[attr] - sample[attr])
                for attr in range(len(sample))
            ])
    return synthetic

# Example usage
minority_data = [[1.0, 2.0], [2.0, 3.0], [3.0, 4.0]]
result = smote(minority_data, N=200, k=2)

Model description original code:
The function is stochastic: it uses random.choice and random.random without a fixed seed, so the exact numbers change every time you run it.  
What can be said for sure is:

• N = 200 ⇒ n_new_samples_per_point = N//100 = 2  
• There are 3 original minority samples ⇒ 3 × 2 = 6 synthetic samples are produced.  
• Each synthetic sample is a 2-dimensional point that lies somewhere on the straight line segment connecting one original point to one of its two nearest neighbours.

So result will always be a list of six 2-element lists, e.g.

[[1.53, 2.53],
 [1.84, 2.84],
 [2.41, 3.41],
 [2.18, 3.18],
 [2.74, 3.74],
 [2.06, 3.06]]

but those numbers will differ on every execution unless the random seed is fixed.
