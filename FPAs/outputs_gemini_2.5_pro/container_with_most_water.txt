def container_with_most_water(height):
    max_area = 0
    left = 0
    right = len(height) - 1

    while left < right:
        width = right - left
        current_height = min(height[left], height[right])
        current_area = width * current_height
        max_area = max(max_area, current_area)

        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
            
    return max_area

heights = [1, 8, 6, 2, 5, 4, 8, 3, 7]
result = container_with_most_water(heights)

#### What the Code Does

This function solves the classic "Container with Most Water" problem. Imagine the numbers in the `height` list are vertical lines. The goal is to find two lines that, when used as the sides of a container, can hold the most water.

The area of water a container can hold is determined by its **width** and the **height of the shorter of the two lines** (since water would spill over the shorter side).

`Area = width * min(height_of_left_line, height_of_right_line)`

#### How the Algorithm Works

The code uses an efficient two-pointer approach:

1.  **Initialization**: It starts with two pointers, `left` at the very beginning of the list and `right` at the very end. This represents the widest possible container.
2.  **Calculate Area**: In each step, it calculates the area formed by the `left` and `right` pointers.
    *   `width = right - left`
    *   `current_height = min(height[left], height[right])`
    *   `current_area = width * current_height`
3.  **Update Maximum**: It keeps track of the largest area found so far in `max_area`.
4.  **Move a Pointer**: This is the clever part. To potentially find a larger area, it needs to find a taller line. Since the width is guaranteed to decrease with every step, the only way to increase the area is by increasing the height.
    *   If the line at the `left` pointer is shorter than the line at the `right` pointer, it moves the `left` pointer one step to the right (`left += 1`). There's no point in moving the taller `right` pointer inward, because the area would still be limited by the short `left` line but with a smaller width.
    *   Otherwise, it moves the `right` pointer one step to the left (`right -= 1`).
5.  **Termination**: The loop continues until the `left` and `right` pointers meet. The function then returns the `max_area` found.
