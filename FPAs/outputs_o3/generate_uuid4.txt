import random

def generate_uuid4():
    rand = random.getrandbits(128)
    # Set version to 4 (UUID version 4)
    rand &= ~(0xf << 76)
    rand |= 0x4 << 76
    # Set variant to 10 (RFC 4122 variant)
    rand &= ~(0x3 << 62)
    rand |= 0x2 << 62
    hex_str = f'{rand:032x}'
    return f'{hex_str[0:8]}-{hex_str[8:12]}-{hex_str[12:16]}-{hex_str[16:20]}-{hex_str[20:32]}'

result = generate_uuid4()

Model description original code:
`generate_uuid4()` returns a randomly generated UUID version-4 string.  
The exact characters change every time you call it, but the format is always:

xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx

• 32 hexadecimal digits in five groups separated by “-”.  
• The first character of the third group is always 4 (the version).  
• The first character of the fourth group is one of 8, 9, a, b (binary 10—the RFC-4122 variant).

So `result` will hold something like:

'3c7d5e2a-fb4d-4c0e-9a3b-2e1f76d4c8e1'

(The concrete value is unpredictable because it depends on the random bits generated at runtime.)
