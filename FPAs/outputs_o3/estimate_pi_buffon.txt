import random
import math

def estimate_pi_buffon(num_trials: int = 100000, needle_length: float = 1.0, line_spacing: float = 1.0) -> float:
    """
    Estimate the value of π using Buffon's Needle experiment.

    Args:
        num_trials: Number of simulated needle drops.
        needle_length: Length of the needle (must be <= line_spacing).
        line_spacing: Distance between two parallel lines on the plane.

    Returns:
        Approximated value of π.
    """
    if needle_length > line_spacing:
        raise ValueError("Needle length must not exceed spacing between lines.")

    hits = 0
    half_spacing = line_spacing / 2.0
    half_length = needle_length / 2.0

    for _ in range(num_trials):
        # Distance from needle center to the nearest line (0 to half the spacing).
        y_center = random.uniform(0, half_spacing)
        # Needle angle with respect to the lines (0 to π/2).
        theta = random.uniform(0, math.pi / 2)
        # Vertical projection of half the needle.
        y_projection = half_length * math.sin(theta)

        if y_center <= y_projection:
            hits += 1

    # Avoid division by zero if no hits occurred (extremely unlikely for reasonable num_trials).
    return float('inf') if hits == 0 else (2 * needle_length * num_trials) / (hits * line_spacing)

# Example usage
result = estimate_pi_buffon(num_trials=50000, needle_length=1.0, line_spacing=1.0)

Model description original code:
`estimate_pi_buffon()` returns a Monte-Carlo estimate of π, so the exact number changes every time you run it.  

With 50 000 trials the estimate is usually within about ±0.01 of the true value 3.14159…  – e.g.

 • one run might give 3.1385  
 • the next 3.1472  
 • another 3.1328

In other words, `result` will hold “something very close to 3.14”.

Model description modified code:
The function performs a Buffon’s-needle simulation.  
With 50 000 trials, a unit-length needle and unit line spacing, the probability of a “hit” is 2 / π ≈ 0.6366, so the expected number of hits is about  

    50 000 × 0.6366 ≈ 31 831.

Plugging that into the returned formula gives  

    (2 · 1 · 50 000) / (31 831 · 1) ≈ 3.1416.

Because the simulation is random, the exact value varies from run to run (typically somewhere around 3.12–3.17 for 50 000 trials).  

Note: as written, nothing is printed; the value ~3.14 is just stored in the variable result.
